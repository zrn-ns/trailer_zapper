# Implementation Tasks

## 1. HTMLに配信サービスモーダルを追加
- `client/index.html` にジャンルフィルターモーダルと同様の構造で配信サービスモーダルを追加
- モーダルのID: `provider-filter-modal`
- 開閉ボタン、バックドロップ、コンテンツエリアを含める
- 6つの配信サービスチェックボックスをモーダル内に配置

## 2. コントロールパネルを簡素化
- `client/index.html` のコントロールパネルから6つのチェックボックスを削除
- 「配信サービスで絞り込み」ボタンを追加（ジャンルボタンと同様のスタイル）
- ボタンのID: `provider-filter-toggle`

## 3. JavaScriptでモーダルの開閉処理を実装
- `client/script.js` に配信サービスモーダルのDOM要素を取得
- モーダルの開閉イベントハンドラーを追加（ジャンルモーダルと同様）
- バックドロップクリックで閉じる処理を追加
- Escキーで閉じる処理を追加（既存のジャンルモーダルと統合）

## 4. モーダル開閉時のUI自動非表示制御を実装
- モーダルが開いている状態を追跡するフラグを追加（例: `isModalOpen`）
- 配信サービスモーダルまたはジャンルフィルターモーダルが開いたときにフラグをtrueに設定
- モーダルが閉じたときにフラグをfalseに設定
- `hideUI()` 関数を修正し、モーダルが開いている場合はUIを非表示にしないように制御
- モーダル開閉時に `isInteracting` フラグも適切に設定
- ビデオエリアクリック時のUI切り替え処理で、モーダルが開いている場合は切り替えを無効化

## 5. 既存のチェックボックス参照を更新
- `client/script.js` の各配信サービスチェックボックスのID参照を更新
- モーダル内の新しいIDに対応させる

## 6. CSSスタイリングを追加
- `client/style.css` に配信サービスモーダルのスタイルを追加
- ジャンルフィルターモーダルと統一されたデザインを適用
- モバイル対応のレスポンシブスタイルを追加

## 7. 動作確認
- すべての配信サービスチェックボックスが正常に機能することを確認
- モーダルの開閉が正常に動作することを確認
- 適用/リセットボタンとの連携が正常に動作することを確認
- モバイル環境でのUIオーバーフローが解消されたことを確認
- モーダル開閉中にUIが自動非表示にならないことを確認
- モーダルを閉じた後、UIが正常に自動非表示になることを確認
