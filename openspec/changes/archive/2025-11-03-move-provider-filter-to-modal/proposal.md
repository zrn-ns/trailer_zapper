# move-provider-filter-to-modal

## Problem
配信サービスのチェックボックス（6つ）がコントロールパネル内に直接表示されており、UIが画面からオーバーフローしています。特にモバイルや小さい画面では、縦スクロールが必要になり、ユーザビリティが低下しています。

## Solution
配信サービスフィルターをジャンルフィルターと同様にモーダルダイアログに移動します。コントロールパネルには「配信サービスで絞り込み」ボタンのみを配置し、クリックするとモーダルが開いてチェックボックスを表示する仕組みにします。

この変更により：
- コントロールパネルがコンパクトになり、画面内に収まるようになります
- ジャンルフィルターと統一されたUXを提供できます
- モバイル環境での操作性が向上します

また、モーダルダイアログを開いている間はオーバーレイUIが自動的に非表示にならないように制御を追加します。これにより、ユーザーがフィルター設定中にUIが消えてしまう問題を防ぎます。

## Scope
- `movie-filtering` spec: 配信サービスフィルターのUI表示方法を修正
- `mobile-ui` spec: モバイル環境でのフィルター操作を改善

## Dependencies
なし

## Breaking Changes
なし（既存の機能は保持され、UIの配置のみが変更されます）

## Alternatives Considered
1. **縦スクロールを許可**: コントロールパネル内にスクロールを追加
   - 却下理由: ユーザビリティが低く、没入感を損なう
2. **アコーディオンで折りたたみ**: 配信サービスセクションを折りたたみ可能にする
   - 却下理由: モーダルの方がより多くのスペースを活用でき、一貫性がある

## Risks
- ユーザーが配信サービスフィルターの場所を見つけにくくなる可能性
  - 緩和策: ボタンのラベルを明確にし、ジャンルフィルターと同じ場所に配置
