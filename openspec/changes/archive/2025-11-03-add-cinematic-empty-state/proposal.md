# add-cinematic-empty-state

## Why

現在、再生できる動画がない場合、`showLoadingMessage()`関数が呼ばれますが、実際にはメッセージを表示せずプレーヤーを空にするだけです。ユーザーは何が起きているのか分からず、次に何をすべきかも不明です。

以下のような状況でこの問題が発生します：
- 配信サービスが選択されていない
- 選択された条件で映画が見つからない
- すべての映画が再生済み
- 予告映像がこれ以上見つからない

映画館の雰囲気を維持しながら、ユーザーに適切なフィードバックと次のアクションを提示することで、よりスムーズなユーザー体験を提供する必要があります。

## What Changes

映画館のスクリーンをイメージしたシネマティックな空状態（empty state）メッセージを表示します。

- 空状態メッセージ用のHTML要素を追加（アイコン、タイトル、メッセージ）
- 暗いスクリーン背景と控えめなテキストで映画館の雰囲気を表現するCSSスタイル
- `showLoadingMessage()`関数を拡張して空状態を表示
- 各エラー状況に応じた適切なメッセージを設定
- ローディング中は空状態を非表示にして雰囲気を維持

デザインコンセプト：
- 暗めの背景（映画館のスクリーン）
- 控えめで映画的なアイコン（フィルムリールやカチンコなど）
- 簡潔で丁寧なメッセージ
- 次のアクションの提案（フィルター変更など）
- 既存のUIデザインとの一貫性

## Impact

- **Affected specs:**
  - `playback-ui` - ADDED: シネマティック空状態表示要件

- **Affected code:**
  - `client/index.html` - 空状態メッセージ要素を追加
  - `client/style.css` - 映画館風の空状態スタイルを追加
  - `client/script.js` - `showLoadingMessage()`関数を拡張、空状態表示ロジックを実装

- **User impact:**
  - 再生できる動画がない場合に明確なフィードバックが得られる
  - 次に何をすべきかが分かりやすくなる
  - 映画館の雰囲気が維持される
